# 연결리스트
- 연결 리스트는 선형 데이터 구조로, 요소가 연속된 위치에 저장되지 않고 포인터를 사용하여 연결됨. 연결리스트는 일련의 연결된 노드를 형성, 각 노드는 데이터와 다음 노드의 주소를 저장.

##### 노드 구조
연결 목록의 노드는 일반적으로 두 가지로 구성.
1. `데이터` : 노드와 관련된 실제 값이나 데이터를 보관.
2. `다음 포인터 또는 참조` : 시퀀스의 다음 노드의 메모리 주소(참조)를 저장.

`Head & Tail` : 연결리스트는 헤드 노드를 통해 액세스하는데, 헤드 노드는 리스트의 첫 번째 노드를 가리킨다. 리스트의 마지막 노드는 NULL 또는 nullptr을 가리키며, 리스트의 끝을 나타냄.

#### 연결리스트의 단점

1. 느린 액세스 시간 : 연결리스트를 탐색하여 찾고 있는 요소를 찾아야 하기 때문에 O(n) 연산으로 빠르게 액세스 하는 상황에 적합하지 않음.
2. 포인터 또는 참조 : 연결리스트는 포인터 또는 참조를 사용하여 다음 노드에 액세스하므로 배열에 비해 사용하기 복잡하다. 이러한 복잡성으로 디버깅하고 유지하기가 어려울 수 있다.
3. 더 높은 오버헤드 : 연결리스트는 배열에 비해 오버헤드가 높다. 연결 리스트의 각 노드는 다음 노드에 대한 참조를 저장하기 위해 추가 메모리가 필요하다.
4. 캐시 비효율성 : 연결 리스트는 메모리가 연속적이지 않기 때문에 캐시 비효율적. 연결리스트를 탐색할 때 캐시에서 필요한 데이털르 얻을 가능성이 낮아 캐시 미스와 느린 성능.
